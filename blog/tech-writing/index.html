技术写作指南

October 10, 2024 · 4 min read
Mason
Full Stack Developer

技术写作是分享知识、建立影响力的重要方式。本文分享技术文章的写作方法和技巧。

🎯 为什么要技术写作​

收益说明
加深理解写作是最好的学习方式
建立影响力展示专业能力
帮助他人回馈社区
职业发展简历加分
沉淀知识个人知识库

📝 写作类型​

1. 教程类​

特点：从零开始，step by step
结构：
- 目标（读者能学到什么）
- 前置条件
- 详细步骤
- 完整代码
- 常见问题

适合：新技术入门、工具使用

2. 原理解析​

特点：深入底层，讲清楚 why
结构：
- 问题背景
- 核心概念
- 实现原理
- 源码分析
- 总结

适合：框架原理、算法解析

3. 最佳实践​

特点：经验总结，实用导向
结构：
- 场景描述
- 常见问题
- 解决方案对比
- 推荐做法
- 注意事项

适合：性能优化、架构设计

4. 踩坑记录​

特点：问题驱动，解决方案
结构：
- 问题描述
- 排查过程
- 根因分析
- 解决方案
- 预防措施

适合：Bug 修复、线上故障

✍️ 写作流程​

1. 选题​

好选题的特征：
- 你真正理解的内容
- 有一定搜索量
- 现有资料不够好
- 你有独特见解

选题来源：
- 工作中的问题
- 学习新技术
- 读书笔记
- 热门话题

2. 大纲​

先写大纲再写内容：
1. 确定核心观点
2. 列出主要章节
3. 每章要点
4. 预估篇幅

示例：
- 引言（200字）
  - 问题背景
  - 文章目标
- 核心内容（2000字）
  - 概念解释
  - 代码示例
  - 对比分析
- 总结（200字）

3. 初稿​

快速写完，不要追求完美：
- 按大纲填充内容
- 代码可以先写伪代码
- 不确定的地方标记 TODO
- 一次性写完，不要中断

4. 修改​

至少修改两遍：

第一遍（结构）：
- 逻辑是否清晰
- 内容是否完整
- 是否有重复

第二遍（细节）：
- 文字通顺
- 代码正确
- 格式规范

💡 写作技巧​

1. 开头吸引人​

❌ 不好的开头：
"本文介绍 React 性能优化"

✅ 好的开头：
"上周我们的 React 应用首屏加载需要 8 秒，用户流失严重。
通过一系列优化，我们将加载时间降到了 2 秒以内。
这篇文章分享我们的优化过程和经验。"

2. 使用例子​

❌ 抽象解释：
"useMemo 用于缓存计算结果"

✅ 具体例子：
"假设你有一个列表需要过滤，每次渲染都重新过滤：

const filtered = items.filter(item => item.active);

如果 items 有 10000 条，每次渲染都会遍历 10000 次。

使用 useMemo 缓存结果，只在 items 变化时重新计算：

const filtered = useMemo(
  () => items.filter(item => item.active),
  [items]
);"

3. 对比说明​

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| Context | 简单 | 性能问题 | 低频更新 |
| Redux | 完善 | 样板多 | 大型项目 |
| Zustand | 轻量 | 生态少 | 中小项目 |

4. 可视化​

使用图表说明：
- 架构图
- 流程图
- 时序图
- 性能对比图

工具推荐：
- draw.io
- Excalidraw
- Mermaid

5. 代码规范​

// ✅ 好的代码示例
// 1. 有注释说明
// 2. 突出重点
// 3. 可以直接运行

// 创建防抖函数
function debounce(fn, delay) {
  let timer = null;
  
  return function (...args) {
    // 清除之前的定时器
    clearTimeout(timer);
    
    // 设置新的定时器
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}

// 使用示例
const debouncedSearch = debounce(search, 300);
input.addEventListener('input', debouncedSearch);

## 核心内容

### 概念2
### 概念3

## 实践/示例

## 总结
- 要点回顾
- 推荐做法
- 延伸阅读

- [ ] 有总结

- [ ] 图片可以显示
